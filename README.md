# Postrelational-DB

---
**Мета роботи**
Розробка UML-діаграми предметної області та проєктування взаємоповʼязаних класів відповідно до вимог. Необхідно було обрати тематику, визначити набір сутностей (класів) та реалізувати між ними різні типи звʼязків. Діаграма повинна містити:
•	спадкування від абстрактного класу;
•	звичайні (single) літеральні властивості;
•	посилання одного класу на інший;
•	використання вбудованого об’єкта;
•	колекції (list або array) простих типів чи об’єктів;
•	символьний або бінарний потік (stream);
•	зв’язки типу one-to-many та parent–children;
•	мінімум одне обов’язкове (required) поле;
•	мінімум одне унікальне (unique) поле;
•	обмеження на значення однієї властивості (min/max або minlen/maxlen);
•	логічний зміст між класами.

---
**Хід виконання**

Було обрано предметну область онлайн-платформи навчання. Система дозволяє організовувати курси, ділити їх на уроки, закріплювати викладачів, а студенти можуть на них навчатись. Кожен урок містить певний тип навчального контенту.

**Типи зв’язку**

Composition (◆)   - 		**`School ◆─ Course`** (курс не існує без школи) <br>
Association			  -     **`Course → Lesson`**, **`Course → Student`**, **`Course → Teacher`** <br>
Inheritance			  -     **`ContentItem ← Lecture/Quiz/Assignment`** <br>
Aggregation/List	-  	  **`Course.lessons: Lesson(), Course.tags: %String()`** <br>
Reference         -     **`Teacher → Phone, Student → Phone, Lesson → ContentItem`** <br>

**Висновки**

У ході виконання роботи було побудовано UML-діаграму предметної області, реалізовано спадкування, референси між класами, колекції, composition-зв’язки та символьний потік. Побудована модель є логічно узгодженою, масштабованою та може бути використана як основа для подальшої реалізації в об’єктно-орієнтованій базі даних InterSystems IRIS.
